# AI Fast Start — Phase 0 Diagnostic

A baseline read on where a business actually sits with AI, and the first win to chase. By Yite Labs.

The diagnostic runs entirely in the browser. The one piece that needs a server is the written read, which is generated by Claude. That call goes through a small serverless function (`app/api/read`) so the Anthropic key stays server-side and never reaches the browser.

## Run it locally

1. `npm install`
2. Copy `.env.example` to `.env.local` and paste in your Anthropic key.
3. `npm run dev`
4. Open http://localhost:3000

## Deploy to Vercel

1. Push this repo to GitHub.
2. In Vercel, "Add New Project" and import the repo.
3. Under Settings → Environment Variables, add:
   - `ANTHROPIC_API_KEY` = your key (get one at https://console.anthropic.com)
4. Deploy. Vercel auto-detects Next.js, no other config needed.
5. Point a domain at it if you like (e.g. diagnostic.yitelabs.co) under Settings → Domains.

The key is only ever read on the server (`process.env.ANTHROPIC_API_KEY`). It is never bundled into the client.

## What costs money

Each completed read is one Claude (Sonnet) call, roughly 1.2k output tokens. The link is open, so there's a simple per-IP rate limit in `app/api/read/route.js` (6 reads per hour per IP) to stop anyone running up a bill. Raise, lower, or replace it with Upstash/Redis if you need something sturdier.

## Tuning the questions

All diagnostic content lives in `lib/diagnostic.js`. Add, remove, or reword questions there. The scoring keys (`weightKey`) decide which lens a question feeds. Questions marked `scoreless` inform the read but don't move the maturity bars. The `DK` option ("don't know") informs the read without counting against the score.

The read's voice and structure live in `lib/prompt.js`.

## Structure

```
app/
  page.js              the diagnostic flow (client component)
  layout.js            root layout + metadata
  components/ui.js     shared presentational components
  api/read/route.js    serverless proxy to Claude (holds the key)
lib/
  diagnostic.js        questions, sections, scoring logic
  prompt.js            builds the system + user prompt in Ben's voice
```
